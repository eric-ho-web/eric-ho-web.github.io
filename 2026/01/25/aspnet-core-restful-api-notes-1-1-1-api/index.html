1.1 API 與 REST

1.1.1 什麼是 API

API，全名為 Application Programming Interface，中文常翻為「應用程式介面」，在開發應用程式時經常會使用到。

簡單來說，API 是一種用來連接兩個系統的介面，讓一方可以透過定義好的方式，向另一方請求服務或資料。

API 的概念

API 可以想成是一個系統「對外公開」的一組操作規格：

  • 系統會定義哪些功能可以被使用
  • 外部應用只能透過 API 與系統互動
  • 不需要知道系統內部是如何實作的

例如，在作業系統上執行的應用程式，會透過作業系統提供的 API 來操作檔案、網路或硬體資源，而不需要直接操作底層實作。

Web API 是什麼

在 Web 應用中，Web API 具備與一般 API 相同的特性，差別在於它是透過 HTTP 對外提供服務。

Web API 通常用來操作資料，例如：

  • 取得資料（查詢）
  • 新增或修改資料
  • 刪除資料

這些 API 可以被各種不同的應用程式呼叫，例如：

  • 網頁前端（如 ASP.NET Core MVC、SPA）
  • 行動裝置應用程式
  • 其他後端系統或服務

跨平台與語言無關

Web API 與平台及程式語言無關：

  • 後端可以使用 Java、.NET、Node.js 等技術實作
  • 呼叫端只要遵守 API 規格即可

例如，一個使用 Java 開發的 Android 應用程式，也可以呼叫使用 C# 撰寫的 ASP.NET Core Web API。

為什麼 API 文件很重要

API 作為一種介面，只對外提供抽象的操作方式，而不暴露其內部實作細節。
呼叫端無須關心 API 是如何實現的，只需要依照規格呼叫對應的方法即可。

Web API 並不是直接給應用程式使用者操作，而是提供給開發人員在開發應用時使用。
因此，一份結構清楚、語意明確的 API 文件，對於前後端協作與系統維護來說非常重要。
