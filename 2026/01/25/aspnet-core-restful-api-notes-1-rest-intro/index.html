第一章 REST 基本概念

REST，全名為 Representational State Transfer，中文常翻為「表述性狀態轉移」，
是一種用來設計 Web 應用程式與 API 的架構風格。

簡單來說，REST 的目標是讓系統：

  • 結構清楚
  • 易於維護
  • 具有良好的效能與擴充性

遵循 REST 設計原則的應用程式，通常被稱為 RESTful 應用。

為什麼要學 REST？

在實務上，大多數 RESTful API 都是基於 HTTP 協議來實作的。
因此，要真正理解 REST，就必須先掌握 HTTP 的核心概念。

本篇文章會介紹什麼？

本系列第一篇將聚焦在 REST 與 API 的基礎概念，內容包含：

  • API 與 REST 的基本定義
  • HTTP 協議核心概念
    • HTTP 方法（GET / POST / PUT / DELETE）
    • HTTP 狀態碼
    • HTTP Headers
    • 媒體類型（Media Type）
  • RESTful API 的設計原則與最佳實踐
  • 實務開發中遇到的問題
    • JSON 與 XML
    • API 版本控制（Versioning）

這些概念將作為後續 ASP.NET Core RESTful API 實作的基礎。
