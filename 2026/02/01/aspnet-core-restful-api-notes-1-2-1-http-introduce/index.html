1.2.1 HTTP 簡介

超文本傳輸（Hyper Text Transfer Protocol, HTTP）協議，
是互聯網上應用最為廣泛的一種網路協議，
也是基於 TCP/IP 協議 的應用層協議。

在日常瀏覽網頁的過程中，
瀏覽器與服務器之間的通信，
正是通過 HTTP 協議 來完成請求與響應的交換，
其整體流程如下所示。

從流程中可以看出，
HTTP 採用的是「請求–響應（Request–Response）」模型。

當 客戶端（通常是瀏覽器） 發起一個 HTTP 請求時，
它會先建立到 HTTP 服務端指定端口
（HTTP 默認使用 80 端口）的 TCP 連接，
而 HTTP 服務器 則負責在該端口監聽來自客戶端的請求。

當 TCP 連接成功建立後，
瀏覽器會向 HTTP 服務器發送請求命令，例如：

一旦服務器接收到請求，
便會根據請求向客戶端返回 HTTP 響應。

HTTP 響應通常包含以下三個部分：

  • 狀態行（例如 HTTP/1.1 200 OK）
  • 消息頭（Headers）
  • 消息正文（Body）

其中，消息正文可能是請求的資源內容、
HTML 文件、JSON 數據、錯誤資訊，或其他相關內容。

需要注意的是，
HTTP 協議本身採用明文方式傳輸數據，
這種通信方式在安全性上存在風險。

因此，網景公司（Netscape） 於 1994 年
設計了 HTTPS（Hypertext Transfer Protocol Secure） 協議，
也常被稱為 HTTP over TLS / HTTP over SSL，
以解決 HTTP 在安全性方面的不足。

在 第 8 章 中，
將對 HTTPS 協議 進行更為詳細的介紹。
