我的 commit 格式

在日常開發中，Git commit 需要能快速被回溯與理解。
這裡記錄目前使用的一套 commit message 結構，用來維持一致且可預期的變更紀錄。

Commit Message Format

Git commit message 採用以下格式：

  • type：標示本次變更的性質
  • scope：標記影響範圍（僅在需要時出現）
  • subject：一句話描述實際變更內容

這個結構讓 commit history 在列表中保持一致的可讀性。

Commit Types in Use

為了降低分類成本，只保留實際長期使用的 commit 類型：

Type 用途描述
feat 新功能相關變更
fix 問題修正
docs 文件調整
style 排版或格式調整（不影響邏輯）
refactor 結構重整，行為維持不變
perf 效能相關修改
test 測試內容變更
chore 雜項、工具或設定調整
revert 回退既有變更

Notes

固定的 Git commit message 格式有助於降低閱讀與理解歷史紀錄的成本。
Commit history 本身即作為變更紀錄存在。

Summary

一致、清楚、可回溯的 Git commit 紀錄方式。
Commit message 依此結構持續書寫。
