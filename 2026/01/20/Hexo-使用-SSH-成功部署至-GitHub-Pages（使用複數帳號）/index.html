Hexo 使用 SSH 成功部署至 GitHub Pages（多帳號情境）

本文記錄在 Windows 環境下，使用 Hexo 將個人網頁部署至 GitHub Pages，
並解決「同一台電腦有多個 GitHub 帳號」時常見的權限與 SSH 身分問題。

一、環境與前提條件

  • 作業系統：Windows
  • 使用工具：
    • Git for Windows（Git Bash）
    • Node.js / npm
    • Hexo
  • 情境：
    • 本機已存在另一個 GitHub 帳號
    • 本次部署使用不同的 GitHub 使用者（username）

二、初始化與靜態檔案產生（確認正常）

在 Hexo 專案根目錄中執行：

hexo g

可看到靜態檔案成功產生至 public/ 目錄。

三、第一次部署錯誤：Deployer not found: git

執行：

hexo d

出現錯誤：

ERROR Deployer not found: git

原因說明：

  • Hexo 本身不包含 git 部署功能
  • 需要額外安裝 deployer 外掛

解法

在專案目錄中執行：

npm install hexo-deployer-git --save

安裝完成後，再次執行：

hexo d

四、Windows Git 安全機制錯誤（safe.directory）

再次部署時出現錯誤：

fatal: detected dubious ownership in repository
F:/Web/PersonalWebsite/.deploy_git

原因：

  • Git 新版在 Windows 上會檢查資料夾擁有權
  • Hexo 自動建立的 .deploy_git 觸發安全機制

解法

執行以下指令（只需一次）：

git config --global --add safe.directory F:/Web/PersonalWebsite/.deploy_git

完成後再次部署：

hexo d

五、權限錯誤：Permission denied（關鍵問題）

部署過程中出現：

ERROR: Permission to username/username.github.io.git denied to other-user.

原因說明：

  • 本機已有另一個 GitHub 帳號的 SSH key
  • Git 預設使用錯誤的 SSH 身分
  • 導致推送至錯誤帳號，被 GitHub 拒絕

六、正確做法：使用 SSH config 指定身分

1. 建立專用 SSH Key（已完成）

例如：

~/.ssh/id_ed25519_web_projects

並將 public key 加入對應的 GitHub 帳號。

2. 設定 ~/.ssh/config

內容如下：

Host github-web
  HostName github.com
  User git
  IdentityFile ~/.ssh/id_ed25519_web_projects

3. 驗證 SSH 身分

執行：

ssh -T git@github-web

成功時會看到：

Hi username! You've successfully authenticated, but GitHub does not provide shell access.

代表 SSH 身分設定正確。

七、修正 Hexo deploy 設定（最重要）

編輯 Hexo 專案根目錄的 _config.yml：

deploy:
  type: git
  repository: git@github-web:username/username.github.io.git
  branch: master

重點說明：

  • 必須使用 github-web(跟你的ssh -T git@”這裡”相符)
  • Host 名稱需與 ~/.ssh/config 完全一致

八、最終部署成功

在專案目錄中執行：

hexo d

成功時會看到：

  • Git commit 成功
  • Git push 完成
  • GitHub Pages 內容更新

瀏覽以下網址即可看到個人網頁：

https://username.github.io

九、補充說明（可忽略警告）

部署時若出現：

LF will be replaced by CRLF

說明：

  • Windows 與 Unix 行尾差異提醒
  • 不影響部署結果
  • 可安全忽略

十、總結

本次部署成功的關鍵在於：

  • 安裝正確的 Hexo deployer
  • 解決 Git Windows 安全目錄限制
  • 明確指定 SSH 使用的 GitHub 身分
  • 在 _config.yml 中使用自訂 Host

此流程適用於：

  • 一台電腦多 GitHub 帳號
  • 多個 Hexo / Web 專案共存
  • 需要精準控制部署身分的情境
